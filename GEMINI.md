# Правила работы с проектом

Ты — мой AI-ассистент. Я пишу дипломный проект, используя связку Laravel (бэкенд) и React (фронтенд).

## Основные правила общения
1. **Язык:** Абсолютно всегда отвечай, пиши объяснения и оставляй комментарии в коде строго на русском языке, даже если логи ошибок или исходный код на английском.
2. **Формат:** Отвечай кратко, по делу, без лишней воды. Фокусируйся на решении задачи.

## Общие правила чистого кода
1. **Паттерн Early Return:** Избегай глубокой вложенности `if-else`. Если есть условие для прерывания функции — возвращай результат сразу в начале.
2. **Никакого хардкода:** Все важные ключи, токены и константы выноси в `.env` или конфигурационные файлы.
3. **Осмысленные комментарии:** Комментируй только неочевидную бизнес-логику (почему это сделано именно так). Не комментируй то, что и так понятно из названия переменных (что делает код).

## Визуальный стиль (Techwear / Cyberpunk)
1. **Эстетика:** Тёмная тема (Dark charcoal #0A0A0B), неоновые акценты (Green #CCFF00 / Cyan #00F0FF).
2. **UI:** Тонкие линии 1px, эффект Glassmorphism (матовое стекло), сетки (grid) на фоне.
3. **Типографика:** Сочетание современных гротесков (Inter) и моноширинных шрифтов (JetBrains Mono) для тех. данных.
4. **Вайб:** Технологично, чисто, футуристично. Больше воздуха, меньше лишних границ.

## Правила для Laravel (Бэкенд)
1. **Валидация:** Всегда используй `FormRequests` для проверки входящих данных. Не пиши логику валидации внутри контроллеров.
2. **Ответы API:** Возвращай данные строго через `API Resources` (Resource Classes), чтобы структура JSON была предсказуемой и скрывала системные поля базы данных.
3. **Работа с БД:** Используй Eloquent ORM. Жестко контролируй проблему «N+1 запроса» — всегда используй жадную загрузку `with()`, если обращаешься к связанным моделям.

## Правила для React (Фронтенд)
1. **Современный подход:** Используй исключительно функциональные компоненты и хуки (Hooks). 
2. **Деструктуризация:** Всегда деструктуризируй `props` прямо в аргументах функции компонента для чистоты и читаемости кода.
3. **Обработка ошибок:** При запросах к Laravel-бэкенду всегда используй блоки `try-catch`. Обязательно обрабатывай статусы ошибок (особенно `422 Unprocessable Entity` для ошибок валидации) и предлагай логику для вывода понятных уведомлений интерфейса.
