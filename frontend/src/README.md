# Структура фронтенда (ВОЯЖРЕЙТ)

## Папки

- **`components/`** — переиспользуемые UI-компоненты
  - `icons/` — SVG-иконки (Icons.tsx)
  - `layout/` — шапка, сайдбар, футер, оверлей поиска
  - `modals/` — модальные окна (редактирование профиля)
  - `RichTextEditor.tsx` — редактор новостей (TipTap)
  - `UI.tsx` — кнопки, аватар, рейтинг, тосты
- **`views/`** — экраны приложения (одна вкладка/раздел = один файл)
- **`services/`** — API-клиент (`apiService.ts`)
- **`utils/`** — утилиты (например, `string.ts` — работа с текстом/HTML)
- **`types.ts`** — общие типы TypeScript
- **`constants.ts`** — константы (URL плейсхолдеров, лимиты)
- **`App.tsx`** — корневой компонент, состояние и маршрутизация по разделам
- **`main.tsx`** — точка входа, Error Boundary

## Поток данных

- Состояние (пользователь, предметы, отзывы, дропы, новости и т.д.) хранится в **App.tsx** и передаётся вниз пропсами.
- Загрузка данных — в `useEffect` в App, запросы через **`apiService`** к бэкенду (`VITE_API_URL`).
- Навигация — через `viewState` и функции `navigateTo` / `setViewState` (без react-router).

## Важно

- Для полей с бэкенда, которые могут быть `null` (например, `article.body`), в коде используется `?? ''` или утилиты вроде `stripHtmlToPlain`, чтобы не было ошибки «Cannot read properties of null».
